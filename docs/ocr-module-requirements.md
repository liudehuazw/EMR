# OCR识别模块产品需求文档

## 📋 项目概述

### 产品名称
电子病历管理系统 - OCR识别模块

### 产品目标
为电子病历管理系统提供高精度的医疗文档OCR识别功能，自动提取关键医疗信息（如就诊日期、检查项目、检验数值等），提升医疗记录数字化效率。

### 核心价值
- **自动化录入**：减少手动录入医疗记录的工作量
- **准确性提升**：通过OCR技术减少人为录入错误
- **效率优化**：批量处理医疗文档，快速数字化
- **数据结构化**：将非结构化文档转换为结构化数据

## 🎯 功能需求

### 1. 核心功能

#### 1.1 文件上传功能
**优先级：P0（最高）**
- **支持格式**：JPG、PNG、BMP图片格式，PDF文档
- **文件大小限制**：单文件最大10MB
- **上传方式**：拖拽上传、点击选择上传
- **批量处理**：支持多文件同时上传（第一阶段限单文件）

#### 1.2 OCR识别功能
**优先级：P0（最高）**
- **识别引擎**：PaddleOCR（后端服务）
- **识别语言**：中文、英文混合识别
- **识别精度**：目标94%以上
- **处理速度**：单文件处理时间<5秒

#### 1.3 结果展示功能
**优先级：P0（最高）**
- **原始文本**：显示OCR识别的完整文本
- **结构化信息**：提取的关键医疗信息
- **置信度显示**：显示识别结果的置信度
- **处理时间**：显示OCR处理耗时

### 2. 医疗场景专项功能

#### 2.1 医疗日期提取
**优先级：P1（高）**
- **支持格式**：YYYY-MM-DD、YYYY年MM月DD日、MM-DD等
- **医疗关键词**：就诊、就医、检查、检验、体检、手术等
- **数据验证**：自动过滤无效日期和未来日期
- **去重处理**：自动去除重复日期

#### 2.2 医疗数值提取
**优先级：P2（中）**
- **检验数值**：提取检验报告中的数值和正常范围
- **单位识别**：自动识别数值单位（mg/dl、mmol/L等）
- **异常标记**：标记超出正常范围的数值

#### 2.3 医疗项目识别
**优先级：P2（中）**
- **检查项目**：识别血常规、尿常规等检查项目名称
- **科室信息**：提取就诊科室信息
- **医生信息**：识别医生姓名和职称

### 3. 交互功能

#### 3.1 结果编辑功能
**优先级：P1（高）**
- **文本编辑**：允许用户修正OCR识别结果
- **信息确认**：用户确认提取的医疗信息
- **批量确认**：支持批量确认多条提取信息

#### 3.2 历史记录功能
**优先级：P2（中）**
- **识别历史**：保存OCR识别历史记录
- **结果搜索**：支持按日期、关键词搜索历史记录
- **重新识别**：支持对历史文件重新识别

## 🏗️ 技术架构

### 系统架构图
```
Frontend (Vue 3)
    ↓ HTTP Request
Spring Boot (Java 17)
    ↓ HTTP Call
Python OCR Service (FastAPI)
    ↓
PaddleOCR Engine
    ↓
Structured Data
```

### 技术栈选择

#### 前端技术栈
- **框架**：Vue 3 + TypeScript
- **UI组件**：Naive UI
- **HTTP客户端**：Axios
- **文件处理**：原生File API

#### 后端技术栈
- **主服务**：Spring Boot 3.x + Java 17
- **OCR服务**：FastAPI + Python 3.8+
- **OCR引擎**：PaddleOCR（pip安装版本）
- **容器化**：Docker + Docker Compose

#### 数据存储
- **临时存储**：内存存储（第一阶段）
- **持久化存储**：MySQL（后续阶段）

## 🔧 部署方案

### 本地开发环境
```bash
# 1. 启动OCR服务（Docker）
docker run -p 8000:8000 ocr-service

# 2. 启动Spring Boot
mvn spring-boot:run

# 3. 启动Vue开发服务器
npm run dev
```

### 服务端口分配
- **前端服务**：http://localhost:5173
- **Spring Boot**：http://localhost:8080
- **OCR服务**：http://localhost:8000

## 🚨 风险评估

### 技术风险
1. **OCR精度**：医疗文档识别精度可能不达预期
   - **缓解措施**：使用PaddleOCR最新版本，针对医疗场景优化
2. **性能问题**：大文件处理可能较慢
   - **缓解措施**：文件大小限制，异步处理
3. **环境依赖**：Python环境配置复杂
   - **缓解措施**：Docker容器化部署

### 业务风险
1. **数据安全**：医疗文档涉及隐私信息
   - **缓解措施**：本地处理，不上传外部服务
2. **用户接受度**：OCR结果可能需要人工校对
   - **缓解措施**：提供编辑功能，逐步优化精度
